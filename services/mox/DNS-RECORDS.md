# DNS Records for alias.id

DNS records are generated by `mox quickstart`. The exact values (especially DKIM public key) depend on the generated key pair.

## Required Records

| Type | Name | Value | Purpose |
|------|------|-------|---------|
| MX | `alias.id` | `10 mail.alias.id.` | Route email to Mox server |
| A | `mail.alias.id` | `<server-ip>` | Mox server IP |
| TXT | `alias.id` | `v=spf1 a mx ip4:<server-ip> -all` | SPF: authorize senders |
| TXT | `dkim1._domainkey.alias.id` | `v=DKIM1; k=rsa; p=<pubkey>` | DKIM: email signing |
| TXT | `_dmarc.alias.id` | `v=DMARC1; p=none; rua=mailto:dmarc@alias.id` | DMARC: monitoring |
| CNAME | `_mta-sts.alias.id` | `mail.alias.id` | MTA-STS |
| TXT | `_smtp._tls.alias.id` | `v=TLSRPTv1; rua=mailto:tlsrpt@alias.id` | TLS reporting |

## Quickstart DNS Output

Run `mox quickstart` to get the exact DNS records for your server:

```bash
docker run --rm -v ./config:/mox/config -v ./data:/mox/data \
  -e MOX_DOCKER=yes -w /mox --network host r.xmox.nl/mox:latest \
  mox quickstart -hostname mail.alias.id admin@alias.id
```

The command prints the DNS records to stdout. Copy them to your DNS provider.

## DKIM Key Rotation

1. Generate a new DKIM key selector (e.g., `dkim2`):
   - Add new selector config in `domains.conf.template`
   - Generate new key: `mox config dkim keygen`
   - Add both `dkim1` and `dkim2` to `Sign` list
2. Publish the new DKIM DNS record (`dkim2._domainkey.alias.id`)
3. Wait 48 hours for DNS propagation
4. Remove old selector (`dkim1`) from `Sign` list, keep in `Selectors` for verification
5. After 7 days, remove old selector entirely and delete old DNS record

## DMARC Escalation Plan

| Phase | Policy | Duration | Purpose |
|-------|--------|----------|---------|
| MVP | `p=none` | First 30 days | Monitor only — collect reports, verify DKIM/SPF alignment |
| Phase 2 | `p=quarantine` | Days 30–90 | Quarantine failures — validate no legitimate mail is affected |
| Phase 3 | `p=reject` | Day 90+ | Full enforcement — reject unauthenticated mail |

Update the `_dmarc.alias.id` TXT record at each phase transition.
